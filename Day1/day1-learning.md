# What is git? 
Git is a distributed version control system that tracks changes in computer files and coordinates work among multiple developers
# what is github?
GitHub is a cloud-based platform where people store, share, and collaborate on code
# The difference between Git and GitHub?
Git is a version control tool used to track changes in files and code, it can work without github
GitHub is an online platform that provides hosting for Git repositories and collaboration features, it requires Git

# Git commands you learned about today
git init – Initialize a new Git repository
git clone – Copy a repository from GitHub to a local machine
git add – Add files to the staging area
git commit – Save changes with a message
git push – Upload local changes to GitHub
git pull – Download the latest changes from GitHub
git status – Check the current status of the repository
git log – View commit history

# Git extensions
git fetch: Downloads the latest changes from the remote repository without merging them
git diff: Shows the differences between files, commits, or branches
git branch: Lists, creates, or manages branches in a repository
git checkout / git switch: Switches from one branch to another
git merge: Combines changes from one branch into another branch
git remote: Manages connections to remote repositories
git stash: Temporarily saves uncommitted changes for later use
git fetch vs git pull: git fetch only downloads changes, while git pull downloads and merges them
Local Repository: The Git repository stored on your computer
Remote Repository: The Git repository hosted on a server
origin: The default name assigned to a remote repository
Tracking Branch: A local branch linked to a remote branch for push and pull operations
